Some Notes on Python

Date Thu 08 January 2026 Tags python / math / object-oriented

TL;DR

Math and Programming overlap in strange ways.

Booleans

Boolean values are a small domain of \(\{\mathtt{True}, \mathtt{False}\}\) and a few operators. The usual operator culprits are and, or, and not. We'll use the math symbols \(\wedge\), \(\vee\), and \(\neg\) for these. Other languages have added other operators, imp, and xor are popular.

To compress the following chart, we'll abbreviate the boolean domain as three values. \(\{\mathtt{T}, \mathtt{F}\}\).

There are 16 possible logic operators. Let's designate these operators with \(p \circledast_n q\). For each operator, the two operands come from \(p = [ \mathtt{F}, \mathtt{F}, \mathtt{T}, \mathtt{T} ]\), and \(q = [ \mathtt{F}, \mathtt{T}, \mathtt{F}, \mathtt{T} ]\). This gives us four argument pairs, and therefore, there will be four possible results.

operator logic notes Python
\(p \circledast_{0} q = [\mathtt{F}, \mathtt{F}, \mathtt{F}, \mathtt{F}]\) \(\text{False}\) Kind of limited. False
\(p \circledast_{1} q = [\mathtt{F}, \mathtt{F}, \mathtt{F}, \mathtt{T}]\) \(p \wedge q\) The and operator. p and q
\(p \circledast_{2} q = [\mathtt{F}, \mathtt{F}, \mathtt{T}, \mathtt{F}]\) \(p \not\Rightarrow q\), or \(p \wedge \neg q\)   not (q or not p)
\(p \circledast_{3} q = [\mathtt{F}, \mathtt{F}, \mathtt{T}, \mathtt{T}]\) \(p\) The value of \(q\) is ignored. p
\(p \circledast_{4} q = [\mathtt{F}, \mathtt{T}, \mathtt{F}, \mathtt{F}]\) \(q \not\Rightarrow p\), or \(q \wedge \neg p\)   not (p or not q)
\(p \circledast_{5} q = [\mathtt{F}, \mathtt{T}, \mathtt{F}, \mathtt{T}]\) \(q\) The value of \(p\) is ignored. q
\(p \circledast_{6} q = [\mathtt{F}, \mathtt{T}, \mathtt{T}, \mathtt{F}]\) \(p \veebar q\), or \(\left(p \vee q\right) \wedge \left(\neg p \vee \neg q\right)\) The exclusive or operator. (p or q) and (not p or not q)
\(p \circledast_{7} q = [\mathtt{F}, \mathtt{T}, \mathtt{T}, \mathtt{T}]\) \(p \vee q\) The or operator. p or q
\(p \circledast_{8} q = [\mathtt{T}, \mathtt{F}, \mathtt{F}, \mathtt{F}]\) \(\neg \left(p \vee q\right)\), or \(\neg p \wedge \neg q\) The "nor" operator. not (p or q)
\(p \circledast_{9} q = [\mathtt{T}, \mathtt{F}, \mathtt{F}, \mathtt{T}]\) \(p \Leftrightarrow q\), or \(\left(p \vee \neg q\right) \wedge \left(q \vee \neg p\right)\) The "biconditional" or "equivalent" operator. (p or not q) and (q or not p)
\(p \circledast_{10} q = [\mathtt{T}, \mathtt{F}, \mathtt{T}, \mathtt{F}]\) \(\neg q\) The value of \(p\) is ignored. not q
\(p \circledast_{11} q = [\mathtt{T}, \mathtt{F}, \mathtt{T}, \mathtt{T}]\) \(q \Rightarrow p\), or \(p \vee \neg q\)   p or not q
\(p \circledast_{12} q = [\mathtt{T}, \mathtt{T}, \mathtt{F}, \mathtt{F}]\) \(\neg p\) The value of \(q\) is ignored. not p
\(p \circledast_{13} q = [\mathtt{T}, \mathtt{T}, \mathtt{F}, \mathtt{T}]\) \(p \Rightarrow q\), or \(q \vee \neg p\) The "conditional" or "implication" operator. q or not p
\(p \circledast_{14} q = [\mathtt{T}, \mathtt{T}, \mathtt{T}, \mathtt{F}]\) \(\neg \left(p \wedge q\right)\), or \(\neg p \vee \neg q\) The "nand" operator. not (p and q)
\(p \circledast_{15} q = [\mathtt{T}, \mathtt{T}, \mathtt{T}, \mathtt{T}]\) \(\text{True}\) The tautology, always true. True

The point of this table is to exhaustively enumerate all 16 possible logic operators. A few of these are available directly. The rest can be implemented using and, or, and not.

(This was not done by hand. I used sympy to help generate this table.)

Numbers

Mathematics offers a variety of number domains.

  • Integers (\(\mathbb{Z}\)). Positive and negative counting numbers, as well as zero: {..., −3, −2, −1, 0, 1, 2, 3, ...}.
  • Rational numbers (\(\mathbb{Q}\)). Numbers that can be expressed as a ratio of an integer to a non-zero integer.
  • Real numbers (\(\mathbb{R}\)). Numbers that correspond to points along a line. They can be positive, negative, or zero. All rational numbers are real, but the converse is not true.
  • Irrational numbers (\(\mathbb {R} \setminus \mathbb {Q}\)). Real numbers that are not rational. These are solutions to polynomials like \(x^2-5=0\).
  • Imaginary numbers: Numbers that equal the product of a real number and the imaginary unit \(j = \sqrt{-1}\), or \(j^2 = 1\).
  • Complex numbers (\(\mathbb{C}\)). Includes real numbers, imaginary numbers, and sums and differences of real and imaginary numbers.

Here are the Python equivalents.

Math Python  
\(\mathbb{Z}\) int  
\(\mathbb{Q}\) from fractions import Fraction  
\(\mathbb{R}\) float is an approximation of real it is not a perfect representation.
\(\mathbb{C}\) complex  

The float type is actually a kind of rational number: \(f = \frac{m}{2^{53}}*2^e\). The upper bound on the denominator leads to truncation. The value of \(2^{53} \approx 10^{15}\), meaning there are about 15 digits of precision available.

Since the value of \(m\), the mantissa, is divided by a power of 2, there will be problems computing a representation that involves any of the prime numbers other than 2. Currency values often involve 0.1 and 0.01, which don't have exact values.

When working with currency, use the decimal module. Import the definition with from decimal import Decimal. The Decimal objects provide real number values that work in base 10. They are slower, consume more memory, but, give precise answers for currency computations, avoiding the complications of binary representation of decimal fractions.
